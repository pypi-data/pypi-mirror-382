*Agent心智模型-上下文系统-构建时上下文[base_agent,数据处理管线,构建时属性]
-settings
在Agent构建的时候就决定了的上下文,用于Agent的静态配置
特点：子级Agent自动覆写继承父级已有的settings
构建方式：e.g. BaseAgent("echo",None,is_debug=True,debug_context=True)
内部支持的键有：
log_dir:str,一般不需要人为定义,日志结果的输出基路径；
对应的辅助函数:from agent_os2 import generate_log_dir,log_dir默认为当前执行路径/memory/statistics_*/
is_debug:bool,对应属性self.is_debug；使用self.debug可输出自定义debug信息
debug_context:bool,是否简化运行时上下文在debug_info.md日志中的记录；
stdout:Callable,该Agent对外的标准输出,默认为print函数
对应的辅助函数:from agent_os2 import record_user_info,参数：agent_instance,info,batch_id,tag,#tag用于外接程序区分信息类型
辅助函数会自动将user_info本地记录至log_dir/user_info.md
stdin:Callable,该Agent对外的标准输入,默认为input函数,被self.get_input函数封装使用
model_config:dict,在Agent未在setup明确指定模型配置时的默认配置,默认为self.model_config的默认值"gemini-2.5-flash"
e.g.{"model_config":{"model_name":"gpt-5",is_stream:True,max_tokens:2048}}
max_*_limit:int|float:用于资源控制
